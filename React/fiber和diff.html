# Fiber 和 Diff

# 1、Fiber

在前端开发中，特别是在处理复杂的用户界面更新时，传统的浏览器渲染机制存在一些性能瓶颈。

比如，当需要对大量DOM节点进行频繁更新时，可能会导致页面卡顿，因为JavaScript执行、布局计算、绘制等操作是依次进行的，一旦有大量的同步更新任务堆积，就会阻塞主线程，影响用户体验。

为了解决这些问题，React引入了Fiber架构。

Fiber本质上是一种对React组件渲染和更新过程进行重新设计的方案，它可以将一个大的更新任务分解成多个小的任务片段，实现可中断、可恢复的异步渲染机制。

# 1、核心特点

  • 可中断与可恢复：Fiber允许渲染任务在执行过程中被中断，然后在合适的时候恢复执行。例如，当一个组件的渲染过程被其他更紧急的任务（如用户输入事件处理）打断时，它可以暂停当前的渲染操作，先去处理紧急任务，处理完后再回来继续完成之前中断的渲染任务。

  • 优先级调度：Fiber为不同的任务分配了不同的优先级。比如，用户输入相关的更新任务通常会被赋予较高的优先级，因为它们直接影响用户当下的交互体验；而一些后台数据更新导致的页面更新任务可能优先级相对较低。通过这种优先级调度，可以确保重要的任务能够及时得到处理，提高整体的用户体验。

  • 任务细化与分批处理：它把一个完整的渲染更新任务分解成许多微小的任务单元，也就是所谓的“Fiber节点”。这些小任务可以按照优先级依次进行处理，并且可以在不同的帧之间进行分配，避免一次性处理大量任务导致主线程长时间阻塞。

# 2、 在React中的实现示例

在React组件的生命周期中，当组件需要更新时，React会创建一系列的Fiber节点来代表组件及其子组件的更新任务。例如，假设有一个包含多个子组件的父组件，当父组件的状态发生变化时：

import React, { useState } from 'react'; 
const ParentComponent = () => {
    const [count, setCount] = useState(0);
    const handleClick = () => {
        setCount(count + 1);
    };
    return (<div>
                <h1>Parent Component</h1>
                <button onClick={handleClick}>Increment</button>
                <ChildComponent count={count} />
            </div>); 
}; 
const ChildComponent = ({ count }) => {
    return (<div>
                <h2>Child Component with count: {count}</h2>
            </div>); 
}; 
export default ParentComponent;

# 2、Diff

在前端开发中，当对用户界面进行更新时，往往需要确定哪些部分发生了变化，以便能够高效地进行更新操作。例如，在一个复杂的网页应用中，可能有大量的DOM元素，每次状态变化后，如果要重新渲染所有元素，效率会非常低。因此，需要一种机制能够快速准确地找出新旧状态下DOM元素之间的差异，这就是Diff算法要解决的问题。

# 1、核心特点

  • 基于树结构的比较：Diff算法通常是基于树状结构（如DOM树或虚拟DOM树）进行比较的。它会将新旧两棵树的节点进行一一对比，找出其中的差异点。

  • 高效的比较策略：为了提高比较效率，Diff算法采用了一些特定的比较策略。比如，在React中使用的Diff算法遵循三个主要原则：

    • 同层比较：只对树的同层节点进行比较，不会跨层比较。也就是说，如果一棵树上某一层的节点发生了变化，它只会在该层内寻找对应的节点进行比较，不会跳到其他层去。

    • 元素类型判断：首先判断节点的元素类型是否相同。如果相同，则继续比较节点的属性和子节点；如果不同，则直接认为该节点发生了重大变化，会将旧节点替换为新节点。

    • 唯一键值标识：当比较子节点时，对于列表型的子节点（如数组形式的子组件列表），要求每个子节点都要有一个唯一的键值（key），以便能够准确地识别每个子节点在新旧状态下的变化情况。

# 2、在React中的实现示例

假设我们有一个简单的React应用，展示一个列表组件，并且可以通过按钮点击来更新列表内容：

import React, { useState } from 'react';

const ListComponent = () => {
    const [items, setItems] = useState(['item1', 'item2', 'item3']);

    const handleClick = () => {
        // 更新列表内容
        setItems(['item2', 'item3', 'item4']);
    };

    return (
        <div>
            <button onClick={handleClick}>Update List</button>
            <ul>
                {items.map((item, index) => (
                    <li key={index}>{item}</li>
                ))}
            </ul>
        </div>
    );
};

export default ListComponent;

当点击 Update List 按钮，改变 items 的值时，React会使用Diff算法来比较新旧状态下的虚拟DOM树（这里假设React内部会构建虚拟DOM树来进行操作）。

首先，它会根据同层比较原则，对 ul 树的同层节点（也就是各个 li 节点）进行比较。

由于我们给每个 li 节点都设置了键值（这里用的是索引 index，实际应用中最好用更有意义的唯一标识），所以能够准确地识别出哪些 li 节点发生了变化（比如，从 item1 变为 item2 的那个节点），然后根据比较结果，对发生变化的节点进行相应的更新操作，而不是重新渲染整个 ul 树。

Fiber和Diff在前端开发中尤其是在React框架的应用场景下，起着非常重要的作用，它们分别从渲染机制优化和高效更新找差异的角度，提升了用户界面的性能和更新效率。
