# dist目录 .git丢失

# 1、原因分析

在 VuePress 打包（yarn docs:build）生成dist目录时，这个目录主要是用于存放可部署的静态文件。

.git目录是用于版本控制的，它包含了项目的版本历史、分支信息等内容。

但是在项目更新后，再次打包时，.git目录会发生丢失，导致无法推送到github。

# 2、解决方案

# 通过.gitignore文件

在项目的根目录下（一般是和package.json同一级），有一个.gitignore文件。如果没有，可以新建一个。

在这个文件中，添加瑞如下内容。

# 忽略dist目录下的.git
dist/.git
1
2

这样打包时，会自动忽略dist目录下可能存在的.git目录。

也可以指定完整目录

docs/.vuepress/dist/.git
1

# 在打包脚本中添加过滤逻辑

如果通过自定义的脚本或者工具来进行打包和构建，可以在脚本中添加逻辑来检查并排除dist目录下的.git目录。

以 Node.js 脚本为例，假设你在scripts目录中有一个自定义的构建脚本build.js，可以使用fs - extra库来检查和删除dist目录下的.git目录。

首先，安装fs - extra：

yarn add fs - extra
1

或

npm install fs - extra
1

然后，在build.js脚本中添加以下逻辑：

const fs = require('fs - extra');
const path = require('path');

async function build() {
  // 假设这里是你的VuePress打包逻辑
  //...

  // 检查并删除dist目录下的.git目录
  const gitDirInDist = path.join(__dirname, 'dist/.git');
  try {
    if (fs.existsSync(gitDirInDist)) {
      await fs.remove(gitDirInDist);
    }
  } catch (error) {
    console.error('Error removing.git from dist:', error);
  }
}

build();
