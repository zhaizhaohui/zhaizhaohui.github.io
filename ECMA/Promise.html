# Promise

Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果。从语法上说，Promise 是一个对象，从它可以获取异步操作的消息。

当使用回调函数来进行事件处理的时候，如果嵌套多层回调函数的时候，就会出现回调地狱，使用Promise就能避免回调地狱的情况。

作用：解决异步回调问题

# 使用

let promise = new Promise(function(resolve,reject){
    // 异步处理
    // 成功调用resolve 往下传递参数 且只接受一个参数
    // 失败调用reject  往下传递参数 且只接受一个参数
})

// Promise回调
// promise回调中then的第二个参数err等同于catch，所以一般只写第一个参数就好
promise.then(res => {
    // 成功时
    console.log(res)
},err => {
    // 失败时
    console.log(err)
}).catch(err => {
    console.log(err)
})

# resolve

Promise.resolve()，方法接收一个参数，将现有的东西转化为一个对象，并会返回一个处于已完成状态的 Promise。在then()方法中使用完成处理函数才能提取该完成态的Promise传递的值。

let promise = Promise.resolve('king');
promise.then((res)=>{
    console.log(res)  //hi
});

# reject

Promise.reject()方法来创建一个已拒绝状态的Promise，在拒绝处理函数中或者catch()方法中才能接受reject()方法传递的值。

let reject = Promise.reject('reject');
reject.catch((res)=>{
    console.log(res)  //reject
})

Promise.all()方法可用来处理多个promise
let p1 = Promise.resolve('cat')
let p2 = Promise.resolve('pig')
let p3 = Promise.resolve('dog')

Promise.all([p1, p2, p3]).then(res => {
    console.log(res)
    // ['cat', 'pig', 'dog']
    // 练习一下我们之前的解构
    let [r1, r2, r3] = res
    console.log(r1)
    // cat
})

若传递给Promise.all() 的某个 Promise 被拒绝了，那么方法所返回的 Promise 就会立刻被拒绝，而不必等待其他的 Promise 结束。

# race

Promise.race()方法与Promise.all()方法相似，不同的是只要有一个Promise是已决状态，那么返回的Promise对象就会立刻成为已决状态。
