# ES6 模块化

在之前的javascript中是没有模块化概念的。如果要进行模块化操作，需要引入第三方的类库。

在ES6之前，社区规定了一套模块规范 Commonjs 主要用于服务端 nodeJs require AMD requireJs, curlJs CMD seaJs

直至ES6提供了通用的模块化方案：import和export

# export

在ES6中每一个模块即是一个文件，在文件中定义的变量，函数，对象在外部是无法获取的。如果你希望外部可以读取模块当中的内容，就必须使用export来对其进行暴露（输出）。

/* test.js */
let name = 'king'
let age = 18
let show = () => {
    // 顺便复习一下字符串模板
    let str = `my name is ${name}, i am ${age} years old`
    console.log(str)
}
// 导出
export {
    name,
    age,
    show as my
}
// 可以通过as来使用别名

// 默认导出
export default {
    job: 'worker'
}

# import

导入外部的模块，并进行使用 import会自动提升到顶部，首先执行

// 在引入的时候也可以通过as来使用别名
improt {name as n, age as a, my} form './test.js'
// 默认导出不需要用{}
improt job form './test.js'
// from 可以是相对路径，也可以是绝对路径
improt {name as n, age as a, my} form 'https://www.xxx.com/test.js'

console.log(n)
// king
console.log(a)
// 18
console.log(my()))
// my name is king, i am 18 years old
console.log(job)
// worker

expor导出的内容如有更改，import中也会跟着变动

// test.js
let a = 100
// 定时更改
setTimeout(() => {
    a = 200
}, 2000)
export {
    a
}

// 引入文件
import {a} from './test.js'
console.log(a)
// 100
setTimeout(() => {
    console.log(a)
    // 200
},3000)

# 动态引入

默认import是不能直接写到if里面去的

// 错误的使用
if (n > 1) {
    import {a} from './test.js'
} else {
    import {a} from './test2.js'
}
// 这样使用会报错

由于import返回对象是一个promise，所以可以使用then

import('./test.js').then(res => {
    console.log(res.a + res.b)
})
// 使用动态的路径
function route(n) {
    switch (a) {
        case: 1,
            return './test.js'
            break
       case: 2
            return './test2.js'
            break
    }
}
import(route(1)).then(res => {
    // ...
})

// 还可以使用promise.all()
Promise.all([
    import('./test.js'),
    import('./test2.js'),
]).then(([m1, m2]) => {
  console.log(m1)
  console.log(m2)  
})
