# em和rem的区别

在CSS中，em和rem都是相对长度单位，但它们在参照基准以及应用场景等方面存在明显区别，以下是具体介绍：

# 1、参照基准不同

# em

em是相对于元素的父元素的字体大小来计算其长度值的单位。也就是说，一个元素所设置的em值对应的实际长度，取决于该元素父元素所设置的字体大小。

例如，若某元素的父元素字体大小被设置为16px，那么当该元素设置为1em时，其实际长度就为16px；倘若父元素字体大小后来被修改为20px，此时该元素的1em则变为20px。

# rem

rem则是相对于根元素（通常是<html>元素）的字体大小来确定其长度值的单位。无论元素处于文档树中的哪个位置，只要根元素的字体大小确定，那么该元素设置的rem值所对应的实际长度就以此为基准，不会因中间父元素字体大小的变化而改变。

比如，当根元素的字体大小设定为16px时，文档中任何元素设置为1rem，其实际长度都为16px。即使该元素所在的某个中间父元素字体大小被调整，其基于rem设置的长度依然以根元素字体大小为准。

# 2、级联影响不同

# em

由于em的长度是基于父元素字体大小，所以在具有多层嵌套结构的文档中，元素的实际长度可能会受到多级父元素字体大小变化的影响，呈现出一种级联变化的效果。

例如，有如下HTML结构：

<html>
    <body>
        <div class="parent1">
            <div class="parent2">
                <div class="child">这是子元素</div>           
            </div>
        </div>
    </body>
</html>

假设最初<html>元素字体大小为16px，<body>元素字体大小未单独设置（继承<html>的字体大小），.parent1元素字体大小设置为20px，.parent2元素字体大小设置为18px。

那么对于.child元素，如果设置为1em，其实际长度将按照.parent2元素的字体大小计算，即为18px。 若后续将.parent2元素的字体大小修改为22px，那么.child元素的1em对应的实际长度也会随之变为22px，这体现了em单位在多层嵌套结构下受父元素字体大小变化影响的级联特性。

# rem

rem的长度只与根元素字体大小相关，所以在文档树中无论元素处于何种嵌套层次，其基于rem设置的长度都不会受到中间父元素字体大小变化的影响，不会出现类似em那样的级联变化情况。

仍以上述HTML结构为例，若根元素<html>字体大小设定为16px，那么.child元素设置为1rem，其实际长度始终为16px。即使中间的.parent1、.parent2等父元素字体大小发生改变，.child元素基于rem设置的长度依然保持不变。

# 3、应用场景不同

# em

  • 基于父元素比例布局：em常用于创建与父元素字体大小成比例关系的布局。例如，在设计一个具有多层嵌套结构的文本区域时，可使用em单位设置段落的缩进、行间距等属性。

    这样，当父元素字体大小发生变化时，这些与文本排版相关的元素尺寸也能相应地按照比例进行调整，保持整个文本区域排版的协调性。

  • 相对尺寸调整：在一些需要根据父元素字体大小灵活调整元素尺寸的场景下也很适用。比如设计一个按钮，希望其大小能随着父元素字体大小的变化而有所改变，就可以使用em单位来设置按钮的宽度、高度等尺寸，使得按钮在不同的父元素字体大小环境下都能保持相对合适的视觉比例。

# rem

  • 全局尺寸统一调整：rem在进行页面全局的布局调整时非常有用。通过改变根元素的字体大小，可以统一调整所有使用rem单位设置的元素的尺寸。

    例如，在设计响应式网页时，根据不同的设备屏幕宽度调整根元素的字体大小，进而实现整个页面元素尺寸的自适应调整。这样可以确保在不同设备上，页面的整体布局和元素比例都能保持相对一致。

  • 避免级联混乱：由于其不受中间父元素字体大小变化的影响，在一些复杂的嵌套结构且不希望因父元素字体大小调整而导致元素尺寸混乱的场景下，rem是更好的选择。

    比如在设计一个大型网站的导航菜单，其中包含多个层级的子菜单，使用rem单位设置菜单元素的尺寸，可以保证无论在哪个层级的父元素字体大小发生变化，菜单元素的尺寸都能保持稳定，不会出现因级联影响而导致的布局错乱情况。
