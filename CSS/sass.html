# Sass

Sass是成熟、稳定、强大的CSS预处理器，而SCSS是Sass3版本当中引入的新语法特性，完全兼容 CSS3 的同时继承了 Sass 强大的动态功能。

# 特性

1、完全兼容 CSS3

2、在 CSS 基础上增加变量、嵌套 (nesting)、混合 (mixins) 等功能

3、通过函数进行颜色值与属性值的运算

4、提供控制指令 (control directives)等高级功能

5、自定义输出格式

# @import

使用 @import 在一个样式文件中引用另一个样式文件，在css中也用，只是不常用。

在css中，只有执行到 @import 时，浏览器才会去下载其他css文件。

在sass中，编译时就会把 @import 的文件导入进来，所有的样式会归纳到生成文件中。

例如：在 my.scss 中使用 @import 引入 color.scss

color.scss 文件

$color-blue: blue;

# 混合器

当网站样式变得越来越复杂，便需要大段的重用样式的代码，独立的变量就没办法应付这种情况了。

混合器使用 @mixin 标识符定义，这个标识符给一大段样式赋予一个名字，这样就可以轻易地通过引用这个名字重用这段样式。

例如定义一段圆角边框的样式

@mixin rounded-corners {
    -moz-border-radius: 5px;
    -webkit-border-radius: 5px;
    border-radius: 5px;
}

# 继承

sass通过@extend语法实现一个选择器可以继承为另一个选择器定义的所有样式。

.base-button {
    font-family: Arial, sans-serif;
    border: 1px solid gray;
    padding: 5px 10px;
}

.primary-button {
    @extend.base-button;
    background-color: blue;
    color: white;
}

.secondary-button {
    @extend.base-button;
    background-color: gray;
    color: black;
}

如果一个选择器继承了多个其他选择器，而且这些选择器之间又有复杂的继承关系，那么在后续修改样式时，可能会出现难以理清头绪的情况。

# 模块化

sass通过 @use 加载样式表

通过 @use 加载的模块不管被引用了多少次，都只会在编译后输出一次到 css 中。

但是使用 @import 多次引入同一模块，会反复输出到 css 中。

@use 'buttons';

    color: buttons.$button-color;

可以通过在 @use 语句中添加 as 关键字来重命名导入模块的命名空间。

@use 'buttons' as btn;

# Script

Sass Script 是 Sass 中用于处理变量、运算、函数、控制指令等的脚本部分。

# 1、数据类型

  • 数字（Numbers）：包括整数和小数，如10、3.14等，可以进行数学运算。

  • 字符串（Strings）：用引号括起来，如'hello'、 "world"。

  • 颜色（Colors）：可以用多种方式表示，如十六进制值（#336699）、RGB 值（rgb(51, 102, 153)）、HSL 值（hsl(210, 50%, 40%)）等。

  • 布尔值（Booleans）：只有true和false两种值，常用于条件判断。

  • 列表（Lists）：可以看作是一组值的集合，用空格或逗号分隔，如10px 20px 30px（空格分隔的长度列表）或(red, green, blue)（逗号分隔的颜色列表）。

  • 映射（Maps）：类似于其他编程语言中的字典，是键值对的集合，例如：

    $font-sizes: (
    small: 12px,
    medium: 16px,
    large: 20px
);

# 输出格式

通过 :style option 选项设定，或者在命令行中使用 --style 选项

1、expanded

Expanded 是默认输出，更像是手写的样式，选择器、属性等各占用一行，属性根据选择器缩进，而选择器不做任何缩进。

2、compressed

Compressed 输出方式删除所有无意义的空格、空白行、以及注释，力求将文件体积压缩到最小，同时也会做出其他调整，比如会自动替换占用空间最小的颜色表达方式。
