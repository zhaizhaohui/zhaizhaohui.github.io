# 2、接口继承

接口除了进行一些类型的设置以外，还可以进行继承的操作

// 定义接口
interface Parent {
    prop1: string,
    prop2: number
}

// 接口继承
interface Child extends Parent {
    porp3: string
}

// 接口调用
const obj: Child = {
    prop1: 'a',
    prop2: 10,
    prop3: 'b'
}

# 3、类的修饰符

  • 可选属性
  • 属性默认值
  • 私有属性
  • 受保护属性
  • 静态成员
class Article {
    // 设置属性默认为public
    title: string
    content: string
    // 设置可选属性使用问号
    author?: string
    // 设置属性默认值
    views = 100
    
    // 私有属性，在子类中是无法使用的
    private subtitle?: string
    // 受保护的属性，只能在Article和其子类中进行访问
    protected thumbs: number
    
    // 静态属性
    static subs: number
    // 私有静态属性, 不可以被子类继承，不可以在被外部调用
   
    private static shortTitle: string
    
    // 只读属性,不可以被修改
    private static readonly brief = 'abc'
    
    // 构造函数，title和content是必选属性
    constructor (title: string, content: string) {
        this.title = title
        this.content = content
        // 调用静态私有属性
        // Article.shortTitle
    }
}

const a = new Article('title', 'content')

# 5、抽象类

通常作为子类的基类去使用的，抽象类中可以有抽象的属性，抽象的方法，包括抽象的存储器，当然也可以具有一些非抽象的属性。

抽象的成员必须在抽象类中才能使用

abstract class Animal {
    // 抽象属性
    abstract name: string,
    // 抽象方法
    abstract makeSound(): void
    // 非抽象方法
    move(): void() {
        console.log('run')
    }
}

class Cat extends Animal {

   // 普通属性
   name: string
   // 存储器设置
   get sound(): string
   // 普通方法
   makeSound(): void
}
   
interface class B {
   age: number
}
   
class Dog implements Animal {

# 7、泛型类

在类的名称后面加一个<T>

class MyClass<T> {
    value: T
    
    constructor (value: T) {
        this.value = value
    }
    
    do (param: T) {
        console.log('I can do', this.value)
    }
}

// 如果传的是一个字符串
const myStr = new MyClass<string>('hello')
myStr.do('cry')

// 如果传的是一个数字
const myNum = new MyClass<number>(10)
myNum.do(10)
