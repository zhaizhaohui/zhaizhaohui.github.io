# 浏览器渲染页面的过程

浏览器渲染页面一般会经过以下几个主要过程：

# 1、解析HTML（构建DOM树）

当浏览器接收到一个网页的HTML文档时，首先会对其进行解析。

浏览器会按照HTML的语法规则，将HTML文档中的标签、属性、文本内容等逐个识别出来。 把这些元素按照它们在文档中的层级关系构建成一棵DOM（Document Object Model）树。

例如，一个简单的HTML页面结构如下：

<html>
    <body>
        <h1>标题</h1>
        <p>段落内容</p>
    </body>
</html>

会被构建成一个包含html节点作为根节点，其下有body节点，body节点下又分别有h1和p节点等的DOM树。

# 2、解析CSS（构建CSSOM树）

浏览器在解析HTML的同时或之后，会开始解析网页中的CSS样式表。

CSS样式表可以是通过<link>标签引入的外部样式表，也可以是在<style>标签内的内部样式表。

浏览器按照CSS的语法规则对样式进行解析，将样式规则根据选择器等进行分类整理，并构建出一棵CSSOM（CSS Object Model）树。

CSSOM树记录了页面中各个元素应该应用的样式信息。

# 3、合并DOM树和CSSOM树（生成渲染树）

在分别构建好DOM树和CSSOM树之后，浏览器会将这两棵树进行合并。

合并的过程主要是根据DOM树中的元素去CSSOM树中查找对应的样式信息，然后将元素及其样式信息组合在一起，形成一棵渲染树（Render Tree）。

渲染树中的节点只包含那些会在页面上实际显示出来的元素，比如<head>中的一些元素（如<meta>等）不会出现在渲染树中，因为它们通常不直接参与页面的可视呈现。

# 4、布局（计算布局）

生成渲染树之后，浏览器需要对渲染树中的各个节点进行布局计算。

主要是确定每个节点在页面中的准确位置（如坐标、大小等）。

这涉及到根据父节点的布局属性以及自身的样式属性等来计算。

例如，如果一个<div>元素设置了宽度为50%且父元素的宽度已知，那么就可以计算出该<div>元素的具体宽度值。

# 5. 绘制（渲染）

在完成布局计算后，浏览器就开始进行绘制操作。

绘制就是按照已经确定的布局位置和样式，将渲染树中的各个节点依次绘制到屏幕上。

绘制过程会涉及到多个步骤，比如先绘制背景色、再绘制边框、最后绘制文本等内容，从而完成整个页面的可视呈现。
