# 用户登录及token验证

在前后端完全分离的情况下，Vue项目中实现token验证大致思路如下：

1、第一次登录的时候，前端调后端的登陆接口，发送用户名和密码

2、后端收到请求，验证用户名和密码，验证成功，就给前端返回一个token

3、前端拿到token，将token存储到localStorage和vuex中，并跳转路由页面。处于安全考虑，我在项目中，将token放在了sessionStorage和vuex中。在客户端的localstorage中存放用户名和密码。

4、前端每次跳转路由，就判断sessionStorage中有无 token，没有就跳转到登录页面，有则跳转到对应路由页面。 在router目录下的index.js文件中添加

// 进行路由守卫
router.beforeEach((to, from, next) => {
    // 判断登录设备，禁止PC端登陆
	if (!/Android|webOS|iPhone|iPod|BlackBerry/i.test(navigator.userAgent))
     {
		window.location.href = 'https://xxx.com/404.html'
      }
	// 如果进入登录或注册页面无需验证直接进入
	if (to.path === '/login' || to.path === '/reg') {
		next()
	} else {
		let token = sessionStorage.Authorization
	// 如果存在accessToken并且不为空，则进行传值比较
		if (token && token !== '') {
			next()
		} else {
			next({path: '/login'})
		}
	}
})

5、每次调后端接口，都要在请求头中加token 在src目录下的main.js文件中添加

axios.interceptors.request.use(function (config) {
	let token = (store.state.Authorization !== '') ? store.state.Authorization : sessionStorage.Authorization
	config.headers.common['token'] = token
	return config
})

axios.defaults.headers.common['token'] = store.state.Authorization
axios.defaults.headers.post['Content-Type'] = 'application/x-www-form-urlencoded'

6、后端判断请求头中有无token，有token，就拿到token并验证token，验证成功就返回数据，验证失败（例如：token过期）就返回401，请求头中没有token也返回401

7、如果前端拿到状态码为401或者用户进行退出登录操作，就清除localStorage中的用户信息并跳转到登录页面。
